5 \section desc Description
6 
7 <em>blockMesh</em> is a meshing engine used to create hex meshes in elementary geometrical shapes:
8 - \ref rprism
9 - \ref spheres
10 - \ref cyl_cone
11 
12 It is included with <em>OpenFOAM</em> and requires that the environment has been properly set. On a default
13 installation, this can be done by executing the following command in the working terminal:
14 
15  . /usr/lib/openfoam/openfoam2006/etc/bashrc
16 
17 It is also possible to create assemblies of these elementary shapes using an input dictionary via the `"Input Dict File"` keyword. For more information, see the [<em>OpenFOAM</em> User Guide](https://www.openfoam.com/documentation/user-guide/4-mesh-generation-and-conversion/4.3-mesh-generation-with-the-blockmesh-utility).

23 Generic code for <em>blockMesh</em>:

37  ...
38  }
39  }

44 where `"<SHAPE>"` can be replaced with `"Block"`, `"Sphere"`, or `"Cylinder/Tapered_Cone"`

46 \section params Parameters
47 
48 \subsection general_params General
49 
50 - <strong>`Input Dict File`</strong>: Specifies the input file, if any `false` indicates no input dict should be used.
51 - <strong>`scaleToMeters`</strong>: Scales all input lengths from the default of meters.To set to millimeters, set to 0.001
52 - <strong>`NumCells`</strong>: Number of cells in each direction [<em>x</em>, <em>y</em>, <em>z</em>].For 40 cells in every direction, set to [40, 40, 40]
53 
54 
55 Optionally, `"Cell_Size"` can be provided within the `"blockMesh Parameters"` block. Then, the number of cells in each direction will be determined with
56 \f$\frac{side length}{cell size}\f$. Note that this will override any specified values for `"NumCells"`.
57 
58 \subsection shape_params Shape Specification
59 
60 
61 The `"blockMesh Parameters"` block should contain <em>exactly one</em> of `"Block Parameters"`, `"Sphere Parameters"`, or `"Cylinder/Tapered_Cone Parameters"`.

66 \subsubsection rprism Rectangular Prisms
67 
68 
69 ![An example rectangular prism created and meshed with blockMesh](blockMsh_Box.png "An example rectangular prism created and meshed with blockMesh")
70 
71  "Block Parameters": {
72  "InitialPoint": [0., 0., 0.],
73  "Length": [1., 1., 1.],
74  "Grading": [1., 1., 1.]
75  }
76 
77 
78 - <strong>`InitialPoint`</strong>: Location of reference point, [<em>x</em>, <em>y</em>, <em>z</em>]For an initial point at the origin, set to [0., 0., 0.]
79 - <strong>`Length`</strong>: Side length in each direction [<em>Lx</em>, <em>Ly</em>, <em>Lz</em>].For a 1x1x1 cube, set to [1., 1., 1.]
80 - <strong>`Grading`</strong>: Ratio between the widths of the ending cell and the starting cell in each direction [<em>Gx</em>, <em>Gy</em>, <em>Gz</em>].For uniform cells, set to [1., 1., 1.]

102 \subsubsection cyl_cone Cylinders/Truncated Cones
103 
104 
105 \note This is written as `"Cylinder/Tapered_Cone"` in the source code, and should be specified as shown in the input file. Nevertheless, what is <em>meant</em> is truncated, not tapered.
106 
107 
108  "Cylinder/Tapered_Cone Parameters": {
109  "Center": [1., 1., 1.],
110  "Radius1": 1.5,
111  "Radius2": 0.6,
112  "Height": 2.5
113  "Grading": [1., 1., 1.]
114  }
115 
116 
117 - <strong>`Center`</strong>: Location of center point, [<em>x</em>, <em>y</em>, <em>z</em>] This refers to the center of the base circle.
118 - <strong>`Radius1`</strong>: The bottom radius of the cylinder/truncated cone. For a cylinder, the two radii should be the same.
119 - <strong>`Radius2`</strong>: The top radius of the cylinder/truncated cone. For a cylinder, the two radii should be the same.
120 - <strong>`Height`</strong>: The height of the cylinder/truncated cone.
121 - <strong>`Grading`</strong>: Ratio between the widths of the ending cell and the starting cell in each direction [<em>Gx</em>, <em>Gy</em>, <em>Gz</em>].For uniform cells, set to [1., 1., 1.]
