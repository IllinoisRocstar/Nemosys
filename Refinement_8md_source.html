3 <em>Adaptive Mesh Refinement</em> is a meshing service designed to apply on-the-fly mesh refinement and coarsening for CFD and finite element modeling simulations based on machine learning and solution field values. This service currently has two modules, one for hexahedral meshes and one for tetrahedral meshes, both of which support user-defined and machine-learning-based refinement and coarsening.

5 <strong>Refinement JSON Template</strong>
6 
7 A mesh generation JSON file contains two sections: `"Mesh File Options"`, where the names of the input and output meshes are specified, and `"Refinement Options"`, where the refinement method and its associated parameters are specified.
8

10  "Program Type": "Refinement",
11  "Mesh File Options": {
12  "Input Mesh File": "unrefined_mesh.vtu",
13  "Output Mesh File": "refined_mesh.vtu"
14  },
15  "Refinement Options": {
16  "Refinement Method": "<REFINEMENT METHOD>",
17  ...
18  }
19  }

21 `"<REFINEMENT METHOD"` can be replaced with one of the refinement methods available within <em>Promesh</em>:
22 - `"Omega_h"`
23 - `"value"`

79 <strong>Metric Sources</strong>
80 
81 - <strong>`Type`</strong>:
82  - <strong>`Constant`</strong>: specifies a uniform size field
83  - <strong>`Variation`</strong>: specifies a size field based on the derivative of `"Tag Name"`, which must be the name of a tag on the elements of the vertices
84  - <strong>`Derivative`</strong>: specifies a size field based on the Hessian of `"Tag Name"`, which must be the name of a tag on the elements of the vertices
85  - <strong>`Given`</strong>: specifies a size field given by `"Tag Name"`, which must be the name of a tag on the vertices
86  - <strong>`Implied`</strong>: specifies the size field implied by the current mesh
87  - <strong>`Curvature`</strong>: specifies a size field based on the curvature of the surface of the mesh
88 
89 - <strong>`Isotropy`</strong>:
90  - <strong>`Anisotropic`</strong>: does not alter the metric source
91  - <strong>`Length`</strong>: turns an anisotropic metric into an isotropic one by choosing the maximum metric in any direction
92  - <strong>`Size`</strong>: turns an anisotropic mesh into an isotropic one by choosing the isotropic mesh that preserves the area/volume of the elements.
93 
94 - <strong>`Scales`</strong>:
95  - <strong>`Absolute`</strong>: prevents scaling of the metric source to compensate for a specific targeted element count
96  - <strong>`Scales`</strong>: allows scaling of the metric source to compensate for a specific targeted element count
97 
98 <strong>Transfer Options</strong>
99 - <strong>`Method`</strong>:
100  - <strong>`Inherit`</strong>: copies values from higher dimensional elements to lower dimensional ones. It is assumed that there exists a tag named `"Name"` for all dimensions
101  - <strong>`Linear Interp`</strong>: linearly interpolates `"Name"`, assumed to be defined on points, to the new mesh
102  - <strong>`Metric`</strong>: interpolates `"Name"`, defined on the vertices, by treating it as a metric tensor
103  - <strong>`Pointwise`</strong>: copies value from the nearest cell on the original mesh
104  - <strong>`Density`</strong>: conserves some quantity when transferring
105  - <strong>`Conserve`</strong>: conserves some quantity when transferring. Requires specifying `"Integal Name"`
