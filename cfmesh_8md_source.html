5 \section desc Description
6 
7 <em>cfMesh</em> is an open-source meshing engine implemented on top of <em>OpenFOAM</em>. <em>Promesh</em> comes with a fully integrated <em>cfMesh</em>-based meshing module. <em>cfMesh</em> can be used to refine specific objects within the mesh in multiple ways: through patches (user-defined groups of surface triangles), through objects (simple geometric shapes that contain the volume to be refined), or by applying refinement to all surfaces or edges in the mesh. <em>cfMesh</em> can also generate boundary layers, which are extruded from the surface toward the interior of the mesh.
8 
9 To enable the capability, the <em>Promesh</em> should be compiled with `ENABLE_CFMSH=ON`.
10 
11 \note <em>cfMesh</em> depends on <em>OpenFOAM</em>, so before starting the compile process make sure to load
12 <em>OpenFOAM</em> environment variables. Depending on the version, <em>OpenFOAM</em> can be loaded by sourcing the
13 bashrc, or cshrc scripts provided in the OpenFoam-x.y/etc/. On a default installation, this can be done by executing
14 the following command in the working terminal:
15 
16  . /usr/lib/openfoam/openfoam2006/etc/bashrc
17 
18 Refer to the <em>OpenFOAM</em> documentation for further instructions. After the <em>OpenFOAM</em> environment is
19 loaded, enable <em>cfMesh</em> build by adding this line to the cmake command:
20 
21  -DENABLE_CFMSH=ON

70 \subsection general_params General
71 
72 Only three things are strictly required to run the <em>cfMesh</em> engine: an input file, an output file name, and a `"MaxCellSize"`. However, if a uniform cell size is not satisfactory, other keywords can be used to specify local refinement options.
73 
74 
75 - <strong>`MaxCellSize`</strong>: Maximum cell size generated in the domain. Required
76 - <strong>`MinCellSize`</strong>: If specified, activates automatic refinement and sets a minimum cell size. Optional

79 \subsection boundary_layers Boundary Layers
80 
81 Boundary layers in <em>cfMesh</em> are extruded from the boundary faces of the mesh toward the interior. Their thickness is controlled by the cell size specified at the boundary and the mesher tends to produce layers of similar thickness to the cell size.
82 
83 
84 - <strong>`BoundaryCellSize`</strong>: Global size for boundary cells.
85 - <strong>`BoundaryCellSizeRefinementThickness`</strong>: Specifies the distance from the boundary at which the specified boundary cell size is still applied
86 - <strong>`BoundaryLayers`</strong>:
87  - <strong>`NLayers`</strong>: The number of layers which will be generated in the mesh. Optional; if not provided, default is either 1 or 0, depending on if boundary layers are to be used or not.
88  - <strong>`ThicknessRatio`</strong>: The ratio between the thickness of two successive layers. Optional; if provided, must be larger than 1. Default is 1.
89  - <strong>`MaxFirstLayerThickness`</strong>: The maximum thickness of the first boundary layer. Optional.
90  - <strong>`AllowDiscontinuity`</strong>: Ensures that the number of layers required for a patch will not spread to other patches in the same layer.
91  - <strong>`PatchBoundaryLayers`</strong>: Local settings for individual patches.
92  - <strong>`PatchName`</strong>:
93  - <strong>`NLayers`</strong>: The number of layers which will be generated in the mesh. Optional; if not provided, default is either 1 or 0.
94  - <strong>`ThicknessRatio`</strong>: The ratio between the thickness of two successive layers. Optional; if provided, must be larger than 1. Default is 1.
95  - <strong>`MaxFirstLayerThickness`</strong>: The maximum thickness of the first boundary layer. Optional.
96  - <strong>`AllowDiscontinuity`</strong>: Ensures that the number of layers required for a patch will not spread to other patches in the same layer.
97 - <strong>`AllowDisconnectedDomains`</strong>: Allows for multi-region meshing, if set to `true`.
98 - <strong>`KeepCellsIntersectingBoundary`</strong>: If this is `true`, cells intersected by the boundary remain part of the template. Default is `false`
99 - <strong>`CheckForGluedMesh`</strong>: If this and "KeepCellsIntersectingBoundary" are `true`, remove cells where distinct parts of the mesh are joined together.

107  - <strong>`Angle`</strong>: Minimum intersection angle for edges to be extracted and included in the new geometry input file.

109 \subsection obj_ref Object Refinements
110 
111 Object refinement is used to specify refinement zones inside the volume. These zones can be boxes, cones, hollow cones, spheres, or lines, and each of those shapes has its own associated parameters to be specified. The level of refinement can be specified in two ways: either by specifying both `"AdditionalRefinementLevels"` and `"RefinementThickness"` or by specifying `"CellSize"`.
112 
113 
114 - <strong>`ObjectRefinements`</strong>:
115  - <strong>`Name`</strong>: Name of the object.
116  - <strong>`Params`</strong>: These must be implemented as specified in the <em>cfMesh</em> manual.
117  - <strong>`type`</strong>: Specification of the primitive geometry for the refinement object. Can be `"box"`, `"cone"`, `"hollowcone"`, `"sphere"`, or `"line"`.
118  - <strong>`refinementThickness`</strong>: The thickness of the refinement zone away from the specified object. Entered as a string, eg `"10"`.
119  - <strong>`additionalRefinementLevels`</strong>: The number of additional refinement levels to be applied within the volume, compared to `"MaxCellSize"`. Entered as a string, eg `"2"`.
120  - <strong>`cellSize`</strong>: Cell size within the refinement object. can be given as a value, such as `7.51`.
121  - <strong>`centre`</strong>: Center of the box or sphere objects. <strong>Note spelling.</strong> Must be present for `"box"` and `"sphere"` types. Entered as string, eg `"(0 0 0)"`.
122  - <strong>`lengthX`</strong>: Side length in the <em>x</em> direction for the box object. Must be present for `"box"` type. Entered as string, eg `"100"`.
123  - <strong>`lengthY`</strong>: Side length in the <em>y</em> direction for the box object. Must be present for `"box"` type. Entered as string, eg `"100"`.
124  - <strong>`lengthZ`</strong>: Side length in the <em>z</em> direction for the box object. Must be present for `"box"` type. Entered as string, eg `"100"`.
125  - <strong>`p0`</strong>: The point at which the radius is `"radius0"` for the cone object. Must be present for `"cone"`, `"hollowcone"`, and `"line"` types. Entered as string, eg `"(-10 0 10)"`.
126  - <strong>`p1`</strong>: The point at which the radius is `"radius1"` for the cone object. Must be present for `"cone"`, `"hollowcone"`, and `"line"` types. Entered as string, eg `"(-10 0 10)"`.
127  - <strong>`radius`</strong>: The radius of the sphere object. Must be present for `"sphere"`. Entered as string, eg `"7"`.
128  - <strong>`radius0`</strong>: The radius at `"p0"` for the cone object. Must be present for `"cone"`. Entered as string, eg `"6"`.
129  - <strong>`radius1`</strong>: The radius at `"p1"` for the cone object. Must be present for `"cone"`. Entered as string, eg `"5"`.
130  - <strong>`radius0_Inner`</strong>: The inner radius at `"p0"` for the hollow cone object. Must be present for `"hollowcone"`. Entered as string, eg `"4"`.
131  - <strong>`radius0_Outer`</strong>: The outer radius at `"p0"` for the hollow cone object. Must be present for `"hollowcone"`. Entered as string, eg `"5"`.
132  - <strong>`radius1_Inner`</strong>: The inner radius at `"p1"` for the hollow cone object. Must be present for `"hollowcone"`. Entered as string, eg `"2"`.
133  - <strong>`radius1_Outer`</strong>: The outer radius at `"p1"` for the hollow cone object. Must be present for `"hollowcone"`. Entered as string, eg `"3"`.

139 Mesh quality is improved via smoothing.

150 \subsection local_ref Local Refinement
151 
152 The local refinement section allows for regions of refinement at the boundary. This can be done in two ways: either by specifying both `"AdditionalRefinementLevels"` and `"RefinementThickness"` or by specifying `"CellSize"`.
153 
154 
155 - <strong>`LocalRefinement`</strong>: Optional
156  - <strong>`PatchName`</strong>: Patch name from the surface mesh. Accepts regex
157  - <strong>`AdditionalRefinementLevels`</strong>: Additional refinement levels to the specified `MaxCellSize`.
158  - <strong>`RefinementThickness`</strong>: Thickness of the refinement zone away from the patch.
159  - <strong>`CellSize`</strong>: Desired cell size for local refinement. Entered as value, eg 2.5
