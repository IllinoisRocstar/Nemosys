8 ## Introduction ##
9 *Promesh* is a modular, extensible
10 resource designed for use in typical application development systems as well as
11 distributed web-services environments. The project focus is providing a
12 framework for robust, automated mesh generation, mesh quality analysis, adaptive
13 mesh refinement, and data transfer between arbitrary meshes. Python bindings to
14 the *Promesh* library can also be enabled.
15

415 To acquire *Promesh*, you can download it from Illinois Rocstar's GitHub or clone

421 ### Build and Install ###
422 
423 > **Warning:** The *Promesh* installation path (`PROMESH_INSTALL_PATH`) should not be
424 > the same as the source location (`PROMESH_PROJECT_PATH`). It is however acceptable that
425 > Promesh and its dependencies be installed in the same directory, such that `PROMESH_INSTALL_PATH`
426 > leads to the same directory as `PROMESH_DEPS_INSTALL_PATH`
427 
428 Set the *Promesh* installation path:
429 ```commandline
430 export PROMESH_INSTALL_PATH=/full/path/to/Promesh/install
431 ```
432 
433 Now, we can compile the *Promesh* library, create its Python bindings, and build
434 other utilities. Note that the current name for the Promesh project as installed from GitHub is Nemosys, so
435 the PROMESH_PROJECT_PATH should be set to the directory into which Nemosys was cloned above:
436 ```commandline
437 cd ${PROMESH_PROJECT_PATH}
438 mkdir build && cd build
439 ```
440 The CMake commad must include a prefix path that indicates the location of the dependencies. The
441 command below can be altered to include or exclude TPLs as needed. Note that if `ENABLE_CFMESH` is
442 `ON`, the *OpenFOAM* environment must be set prior to running the cmake command.
443 
444 See the build options tabulated below for more information on the available CMake options.
445 
446 | If your system is... | use the command... |
447 |----------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
448 | Ubuntu | `cmake .. -DCMAKE_PREFIX_PATH="${PROMESH_DEPS_INSTALL_PATH}/opencascade;${PROMESH_DEPS_INSTALL_PATH}/vtk;${PROMESH_DEPS_INSTALL_PATH}/netgen" -DCMAKE_INSTALL_PREFIX=${PROMESH_INSTALL_PATH}/Promesh -DENABLE_TESTING=ON -DBUILD_SHARED_LIBS=ON -DCMAKE_BUILD_TYPE=Release ` |
449 | CentOS | `cmake3 .. -DCMAKE_PREFIX_PATH="${PROMESH_DEPS_INSTALL_PATH}/opencascade;${PROMESH_DEPS_INSTALL_PATH}/vtk;${PROMESH_DEPS_INSTALL_PATH}/netgen" -DCMAKE_INSTALL_PREFIX=${PROMESH_INSTALL_PATH}/Promesh -DENABLE_TESTING=ON -DBUILD_SHARED_LIBS=ON -DCMAKE_BUILD_TYPE=Release ` |
450 
451 To complete the installation:
452 ```commandline
453 make -j$(nproc)
454 make install
455 ```
456 
457 
458 Executing the commands above will build all libraries, executables, and
459 bindings. The libraries are installed in `$PROMESH_INSTALL_PATH/lib`.
460 Executables are installed in `$PROMESH_INSTALL_PATH/bin`. If Python
461 bindings are enabled, the `pyNemosys` module is installed for the user. The
462 `pyNemosys` module can be imported in Python as `import pyNemosys`. The build
463 configuration can be modified through the CMake Curses interface, `ccmake`, or
464 by passing the command line options to `cmake`.

498 **Enabling cfMesh**
499 
500 *cfMesh* is an open-source meshing engine implemented on top of *OpenFOAM*.
501 *Promesh* comes with a fully integrated *cfMesh*-based meshing module. To enable
502 the capability, *Promesh* should be compiled with `ENABLE_CFMSH=ON`.
503 
504 > **Note:** *cfMesh* depends on *OpenFOAM*, so before starting the compile
505 > process make sure to load *OpenFOAM* environment variables. Depending on
506 > the version, *OpenFOAM* can be loaded by sourcing the bashrc, or cshrc
507 > scripts provided in the `OpenFoam-x.y/etc/`. On a default installation, this can
508 > be done with
509 > ```commandline
510 > . /usr/lib/openfoam/openfoam2006/etc/bashrc
511 > ```
512 Refer to the *OpenFOAM* documentation for further instructions. After the *OpenFOAM*
513 environment is loaded, enable a *cfMesh* build by adding this line to the cmake
514 command:
515 ```commandline
516 -DENABLE_CFMSH=ON -DENABLE_GPL=ON
517 ```
518 
519 
520 **Enabling CUDA for Omega_h**
521 
522 Omega_h can be built with CUDA support using the Kokkos backend, assuming Kokkos
523 is built with CUDA support. Currently, only Kokkos version 2 is supported.
524 To enable this, make sure that the following flag is set:
525 ```commandline
526 -DENABLE_OMEGAH_CUDA=ON
527 ```
528 and that `CMAKE_PREFIX_PATH` (or `$PATH`) contains
529 `${PROMESH_DEPS_INSTALL_PATH}/kokkos/lib/CMake`. Note that both Kokkos and
530 *Promesh* must be built as shared libraries (`-DBUILD_SHARED_LIBS=ON`).

536 ```commandline

556 boost. An archive of pre-built Windows dependencies is available with some
557 exceptions:
558 
559 > **Note:** When downloading pre-built libraries, ensure to select the version
560 matching the bitness (32 or 64) and the MSVC compiler version
561 (14.1, 14.0, etc.) used.
562 
563 **boost**: https://sourceforge.net/projects/boost/files/boost-binaries/
564 
565 **netCDF** (only if Exodus/Epic support is enabled): https://www.unidata.ucar.edu/downloads/netcdf/index.jsp
566 
567 **SWIG** (only if Python bindings are enabled): http://swig.org/download.html
568 
569 ### boost Dependency ###
570 
571 boost will install by default to `C:\local\boost_#_##_#` where the `#` are
572 the boost version. The boost location must be specified to CMake with the
573 `BOOST_ROOT` variable.
574 
575 ### netCDF Dependency ###
576 
577 The netCDF installation is not detected automatically. Pass the location to the
578 build system through the `CMAKE_PREFIX_PATH` variable.
579 
580 ### Archived Dependencies ###
581 
582 Extract the archive to a custom location (`%TPL_DIR%). The dependencies must
583 be given to CMake through the `CMAKE_PREFIX_PATH` variable.
584 ```
585 -DCMAKE_PREFIX_PATH="%TPL_DIR%\cgns;%TPL_DIR%\exodusii;%TPL_DIR%\gmsh;%TPL_DIR%\hdf5\cmake\hdf5;%TPL_DIR%\metis;%TPL_DIR%\netgen;%TPL_DIR%\vtk;%TPL_DIR%\zlib"
586 ```
587 
588 Exclude any you wish to use a custom installation.
589 
590 ### Python on Windows ###
591 
592 Python bindings are generated using SWIG. A SWIG installer is needed on the
593 system and must be available to CMake and available to the Python environment.
594 
595 This is accomplished in two parts:
596 1. Add the `SWIG_EXECUTABLE` variable pointing to the SWIG executable (`swig.exe`) for
597  CMake.
598 2. Add the folder containing the SWIG executable to the system `PATH` for Python.
599 
600 A pre-built Windows executable can be downloaded from the
601 [SWIG website](http://swig.org/download.html)
602 
603 Note: On a system with multiple Python installations, the Cmake variable
604 `PYTHON_EXECUTABLE` can be set to the `python.exe` executable to force the
605 specific environment.

608 ### Build Promesh ###
609 
610 With the dependencies specified above installed, we can compile *Promesh* with
611 Python bindings and build tools with the following command from a MSVC command
612 prompt:
613 
614 ```
615 > set NETCDF_LOCATION=C:\full\path\to\netCDF\install
616 > set BOOST_LOCATION=C:\full\path\to\boost\install
617 > set SWIG_EXE=C:\full\path\to\SWIG\executable\swig.exe
618 > set PROMESH_INSTALL_PATH=C:\full\path\to\Promesh\install
619 > cd %PROMESH_PROJECT_PATH%
620 > md build && cd build
621 > cmake .. ^
622  -G "Ninja" ^
623  -DBOOST_ROOT="%BOOST_LOCATION%" ^
624  -DSWIG_EXECUTABLE="%SWIG_EXE_LOCATION%" ^
625  -DCMAKE_INSTALL_PREFIX="%PROMESH_INSTALL_PATH%" ^
626  -DBUILD_SHARED_LIBS=ON ^
627  -DENABLE_TESTING=ON ^
628  -DENABLE_BUILD_UTILS=ON ^
629  -DENABLE_PYTHON_BINDINGS=ON ^
630  -DCMAKE_PREFIX_PATH="%TPL_DIR%\cgns;%TPL_DIR%\exodusii;%TPL_DIR%\gmsh;%TPL_DIR%\hdf5\cmake\hdf5;%TPL_DIR%\metis;%TPL_DIR%\netgen;%TPL_DIR%\vtk;%TPL_DIR%\zlib;%NETCDF_LOCATION%"
631 > ninja -j %NUMBER_OF_PROCESSORS% (or however many threads you'd like to use)
632 > ninja install
633 ```
634 Executing the commands above will build all libraries, executables, and
635 bindings. The libraries are installed in `%PROMESH_INSTALL_PATH%\lib`.
636 Executables are installed in `%PROMESH_INSTALL_PATH%\bin`.

650 ## Running Promesh ##
651 
652 ### Setting the Environment ###
653 
654 Using *Promesh* requires setting the environment. In most cases, this will be
655 done by setting the `LD_LIBRARY_PATH` in the terminal from which *Promesh* cases
656 will be run, via a command like the one below:
657 
658 ```commandline
659 export PROMESH_DEPS_INSTALL_PATH=/path/to/dependencies/installation/path
660 
661 export LD_LIBRARY_PATH=${PROMESH_DEPS_INSTALL_PATH}/vtk/lib:${LD_LIBRARY_PATH}
662 export LD_LIBRARY_PATH=${PROMESH_DEPS_INSTALL_PATH}/opencascade/lib:${LD_LIBRARY_PATH}
663 export LD_LIBRARY_PATH=${PROMESH_DEPS_INSTALL_PATH}/netgen/lib:${LD_LIBRARY_PATH}
664 ```
665 
666 For the GPL licensed version of *Promesh*, additional library paths must be set for
667 *Gmsh* and *OpenFOAM*:
668 ```commandline
669 export LD_LIBRARY_PATH=${PROMESH_DEPS_INSTALL_PATH}/gmsh/lib:${LD_LIBRARY_PATH}
670 export LD_LIBRARY_PATH=/usr/lib/openfoam/openfoam2006/platforms/linux64GccDPInt32Opt/lib/openmpi-system:${LD_LIBRARY_PATH}
671 ```
672 
673 The *OpenFOAM* environment must also be set:
674 ```commandline
675 . /usr/lib/openfoam/openfoam2006/etc/bashrc
676 ```
