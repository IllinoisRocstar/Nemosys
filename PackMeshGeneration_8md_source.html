1 @page packmesh_ref Pack Mesh Generation
2 
3 There are three workflows embedded within the <em>Pack Mesh</em> engine to generate 3D hex(-dominant) multi-material meshes, 3D periodic tetrahedral multi-material meshes, and 2D surface meshes using input pack geometries in the form of an stl file or a <em>Rocpack</em> geometry file. The basic workflow is as follows:
4 - Generate a background volume mesh.
5 - Snap the extracted surfaces onto the background mesh to create conformal interfaces.
6 - Split the regions into different `cellZones`.
7 - Merge all the different volumes into two foam meshes: the packs and the surrounding medium.
8 - Create patches for the multiple pack regions.
9 - Convert the foam mesh to VTK.
10 - Write mesh quality statistics to a text file.

49 \subsubsection hex_mesh_files Mesh Files
50 
51 Four files are referenced in the `"Mesh File Options"` section of the pack mesh generation JSON input file:
52  - <strong>`Input Rocpack File`</strong>: the <em>Rocpack</em> file from which the pack mesh will be generated. If using an stl file, instead specify with <strong>`Input Geometry File`</strong>
53  - <strong>`Output Pack Mesh File`</strong>: the mesh of the packed objects.
54  - <strong>`Output Surrounding Mesh File`</strong>: the mesh of the surrounding medium.
55  - <strong>`Output Combined Mesh File`</strong>: the combined pack and surrounding medium meshes.

57  \subsubsection hex_params Hexahedral Pack Mesh Options
58 
59 
60 - <strong>`Type`</strong>: One of `"surface"` or `"hexahedral"`. Required
61 - <strong>`Engine`</strong>: `"packmesh"` Required
62 - <strong>`snappyHexMesh Parameters`</strong>: Required
63  - <strong>`Castellated Mesh`</strong>: If `true`, enables castellated mesh option. Required
64  - <strong>`Snapping`</strong>: If `true`, enables surface snapping option. Required
65  - <strong>`Layer Addition`</strong>: If `true`, enables adding layers. Required
66  - <strong>`Geometry Definition`</strong>: Required
67  `Enable Multi Patches`</strong>: Should be enabled if the geometry file has more than one patch. Required
68  `Input Patch Name`</strong>: If specified, will be assigned to the entire surface of the provided geometry. Required
69  - <strong>`Castellated Mesh Controls`</strong>: Required
70  `CellZones`</strong>: If `true`, indicates that there will be multiple disconnected regions within the mesh. Required
71  `RegionRefine`</strong>: If `true`, indicates additional refinement regions. Required
72  `SurfaceRefine`</strong>: If `true`, indicates additional refinement surfaces. Required
73 - <strong>`blockMesh Parameters`</strong>: Required
74  - <strong>`Input Dict File`</strong>: Specifies the input file, if any. `false` indicates no input dict should be used. Required
75  - <strong>`scaleToMeters`</strong>: Scales all input lengths from the default of meters.To set to millimeters, set to 0.001
76  - <strong>`NumCells`</strong>: Number of cells in each direction [<em>x</em>, <em>y</em>, <em>z</em>].For 40 cells in every direction, set to [40, 40, 40]
77  - <strong>`Block Parameters`</strong>: Required
78  `InitialPoint`</strong>: Location of reference point, [<em>x</em>, <em>y</em>, <em>z</em>] Required if not setting `"Auto_Generate"`
79  `Length`</strong>: Side length in each direction [<em>Lx</em>, <em>Ly</em>, <em>Lz</em>]. Required if not setting `"Auto_Generate"`
80  `Auto_Generate`</strong>: Automatically generate the bounding box. Required if `"InitialPoint"` and `"Length"` are not specified
81  `Offset`</strong>: Required if `"InitialPoint"` and `"Length"` are not specified
82 - <strong>`MeshManipulation mergeMeshes Parameters`</strong>: Options for combining the pack geometries to bring them under one mesh. Optional
83  - <strong>`Master Region Path`</strong>: Specifies the input file, if any. `false` indicates no input dict should be used. Optional
84  - <strong>`Add Region Path`</strong>: Specifies the input file, if any. `false` indicates no input dict should be used. Optional
85  - <strong>`overwrite?`</strong>: Should always be set to `true` for multi-material meshing. Optional
86 - <strong>`MeshManipulation createPatch Parameters`</strong>: Sets names for the patches used. Optional
87  - <strong>`Surrounding PatchName`</strong>: Specifies the name for the surrounding medium. Optional
88  - <strong>`Packs PatchName`</strong>: Specifies the name for the packed material. Optional
89  - <strong>`Surrounding PatchType`</strong>: Specifies the patch type (either `patch` or `wall`) for the surrounding medium. Optional
90  - <strong>`Packs PatchType`</strong>: Specifies the patch type (either `patch` or `wall`) for the packed material. Optional
91  - <strong>`overwrite?`</strong>: Should always be set to `true` for multi-material meshing. Optional

104  \subsubsection surface_params Surface Pack Mesh Options
105 
106 
107 - <strong>`Type`</strong>: One of `"surface"` or `"hexahedral"`. Required
108 - <strong>`Engine`</strong>: `"packmesh"` Required
109 - <strong>`Periodic 3D Mesh`</strong>: Required
110  - <strong>`Physical Group Options`</strong>: One of `"None"`, `"Multi Physical Groups"`, `"Two Physical Groups"`, or `"Physical Group per Shape"` Optional; default is `"None"`
111  - <strong>`Create cohesive elements`</strong>: If `true`, creates zero-thickness 3D elements between the pack and the surrounding region. Optional; default is `false`
112  - <strong>`Enable Patches`</strong>: If `true`, enables writing of surface patches in the final mesh. Optional; default is `false`
113  - <strong>`Set Periodic Geometry`</strong>: Optional; default is `false`
114  - <strong>`Element Order`</strong>: Either 1 or 2 Optional; default is 1
115  - <strong>`Custom Domain`</strong>: Optional
116  `Initial`</strong>:
117  `Length`</strong>:
118  - <strong>`TransferMesh`</strong>: Specifies translation in the [<em>x</em>, <em>y</em>, <em>z</em>] directions. Optional
119 - <strong>`Mesh Size`</strong>: Optional
120 - <strong>`Mesh Algorithm`</strong>: Options are 1, 2, 5, 6, 7, 8, or 9 for 2D meshes and 1, 4, 7, 9, or 10 for 3D meshes; see below. Optional; default is 1
121 - <strong>`Scale Value`</strong>: Sets the amount of reduction, from 0 to 1, with 1 being no scaling Optional
122 - <strong>`Remove geometries on boundary`</strong>: If `true`, any geometry intersecting the boundary will be removed Optional; default is `false`
123 - <strong>`Enable Default Outputs`</strong>: Optional; default is `false`
124 - <strong>`Enable Size Preservation`</strong>: If `true`, pack size is preserved instead of packing fraction. Optional; default is `false`
125 - <strong>`Refinement Levels`</strong>: Refinement applied to the original mesh. Optional
126 - <strong>`Upper Threshold`</strong>: Upper threshold for filtering with respect to mean. Optional
127 - <strong>`Lower Threshold`</strong> Lower threshold for filtering with respect to mean. Optional
