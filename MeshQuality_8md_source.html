4 Currently, <em>Promesh</em> uses <em>cfMesh</em> as a mesh quality optimizer. Because it is based on <em>openFOAM</em>, it assumes the folders that contain the mesh is present in the current directory. Therefore, there are no `"Mesh File Options"` and no mesh input or output files specified in JSON input files for `"Mesh Quality"` programs.

6 <strong>Mesh Quality JSON Template</strong>
7 
8 A mesh quality JSON file contains a single section: `"Mesh Quality Options"`, where the meshing engine and its associated parameters are specified.

10  {
11  "Program Type": "Mesh Quality",
12  "Mesh Quality Options": {
13  "Mesh Quality Engine": "cfmesh",
14  "Schedule": [
15  {
16  "Method": "meshOptimizer",
17  "NIterations": 4,
18  "NLoops": 4,
19  "QualityThreshold": 0.3,
20  "NSurfaceIterations": 4,
21  "ConstrainedCellSet": null
22  }
23  ]
24  }

28 Mesh quality is improved via smoothing.
29 
30 - <strong>`Method`</strong>: Should be set to `"meshOptimizer"`
31  - <strong>`NIterations`</strong>: Number of optimization iterations. Default is 50
32  - <strong>`NLoops`</strong>: Number of inner loops in optimization. Default is 10
33  - <strong>`QualityThreshold`</strong>: Minimum mesh quality before optimization is considered done. Default is 0.1
34  - <strong>`NSurfaceIterations`</strong>: Number of surface iterations. Default is 2
35  - <strong>`ConstrainedCellsSet`</strong>: Name of constrained cell set.
